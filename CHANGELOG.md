## Project Setup - April 2026

**Status:** Planning phase complete

### Documents created
- `LangTap_Planning.md`: Full product vision, feature detail, phase roadmap, audio
  design decisions, onboarding flow, leaderboard structure, and reference projects.
- `LangTap_Sprints.md`: Sprint board covering 11 Phase 1 sprints and Phase 2-4
  backlogs. All tasks sized and noted.
- `CLAUDE.md`: AI session rules, safety safeguards, token optimisation settings,
  strategic compaction guidance, pre-task checklist, session protocol, and
  community references.
- `docs/GAME_DESIGN.md`: Full engine specification covering mastery system,
  per-character first-attempt scoring, reattempt model, word counter system,
  kana unlocking progression (seion, dakuon, yoon all grouped), character
  selection algorithm, input modes, distance mechanic, sokuon and long vowel
  mark handling, and full constants reference.
- `docs/CONTENT.md`: Character data schema, special character specification
  (sokuon and long vowel mark), romaji variant map, word bank schema and
  automated build pipeline, audio asset inventory (word-level only via Kanji
  Alive CC BY 4.0), mnemonic guidelines, credits requirements, and content
  maintenance rules.
- `CHANGELOG.md`: This file.

### Key decisions recorded
- Audio is word-level only. No isolated character sounds. Hearing the full word
  provides better phonetic context than isolated phonemes.
- Mastery scoring is per-character, first-attempt only. Reattempts award no points.
- Sokuon and long vowel mark are full characters in the system with mastery scores,
  Dojo entries, and unlock thresholds.
- Word bank is generated automatically from JMdict via build script. Not hand-built.
- Em-dashes are prohibited across all documents and UI copy.
- Tofugu recommended as a starting point for complete beginners on the landing page.
